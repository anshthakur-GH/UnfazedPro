## Project Name
**Antigravity Intelligence Dashboard**
*(named after the monitoring system: `Antigravity.exe`)*

---

## What You Are Building

A single-page, fully interactive HTML+CSS+JS dashboard website that visualises data from a local workforce activity monitoring system. The data comes from PDF reports generated by a Python monitoring tool called **Antigravity**. The dashboard must be hardcoded with realistic sample data derived from the actual report structure — no backend required, no API calls, pure frontend.

This dashboard will be used in a hackathon demo to a panel of judges. It must be **stunning, futuristic, and data-dense** — the kind of interface that makes judges stop talking and stare at the screen.

---

## Aesthetic Direction — NON-NEGOTIABLE

**Theme:** Dark, cinematic, command-centre aesthetic. Think: NASA mission control meets Bloomberg Terminal meets a sci-fi intelligence briefing room. Not purple gradient startup slop. Not minimal white SaaS. This is a surveillance and intelligence tool — it should feel powerful, serious, and slightly intimidating.

**Color palette:**
- Background: `#020408` (near-black with a cold blue undertone)
- Surface: `#080f1a` (dark navy card background)
- Border/grid lines: `#0d2135` (very dark teal-blue)
- Primary accent: `#00d4ff` (electric cyan — used for glows, highlights, active states)
- Secondary accent: `#00ff88` (neon green — used for positive values, ROI, productive time)
- Warning/distraction: `#ff4444` (harsh red — used for distraction, idle, wasted time)
- Amber: `#ffaa00` (for neutral/medium priority items)
- Text primary: `#e8f4f8` (near-white, slightly cool)
- Text secondary: `#4a7a99` (muted teal-grey)
- Text tertiary: `#1e4060` (very muted, for de-emphasised labels)

**Typography:**
- Display/headings: `'Orbitron'` from Google Fonts — geometric, technical, space-age
- Body/data: `'JetBrains Mono'` from Google Fonts — monospaced, terminal feel, perfect for data
- Labels/captions: `'Rajdhani'` from Google Fonts — condensed, military-esque

**Visual effects required:**
- Scanline overlay across the entire background (CSS repeating-linear-gradient, 2px lines, 3% opacity)
- Subtle animated grid in the background (CSS keyframe animation, slow drift)
- Glowing borders on all cards using `box-shadow: 0 0 20px rgba(0, 212, 255, 0.15), inset 0 0 20px rgba(0, 212, 255, 0.03)`
- Animated pulse dots on "live" session indicators (green dot, CSS animation)
- Number counter animations on all stats (count up from 0 on page load, 1.5s duration)
- Staggered fade-in for all cards on load (animation-delay increasing by 0.1s per card)
- Hover states on all cards: border glow intensifies, slight scale(1.01) transform
- Chart bars and lines animate in on load (CSS transitions, 0.8s ease-out)
- Cursor: custom crosshair or default — no pointer emoji nonsense

## Implementation Rules

1. **Single HTML file.** All CSS and JS inline. No external files except Google Fonts CDN.
2. **No chart libraries.** All charts built with pure CSS (bars, donuts) and SVG (arc gauge, flow diagram). This shows technical skill to judges and loads instantly.
3. **All data hardcoded** from the `DASHBOARD_DATA` object above. JS renders everything dynamically from this object so it looks data-driven.
4. **Counter animations.** Every number stat counts up from 0 using `requestAnimationFrame`.
5. **Boot sequence.** 1.5s boot animation before dashboard appears.
6. **Fully responsive** down to 1024px minimum width (this will be demoed on a laptop).
7. **Custom scrollbar** on all scrollable containers: thin, dark, cyan thumb.
